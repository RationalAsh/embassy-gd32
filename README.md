# embassy-gd32

`embassy-gd32` is an experimental GD32 support crate aimed at Embassy-based projects.

## Status

This repository is still a work in progress.

It should be read as an early Embassy integration effort built on top of existing GD32 peripheral access crates, not as a from-scratch implementation of the full device support surface. Expect rough edges, missing pieces, and APIs that may change as support matures.

## Attribution And Provenance

The bulk of the code in this repository, especially the `src/gd32*` device modules and much of the PAC support layer, is derived from the crates generated by [`gd32-rs`](https://github.com/gd32-rust/gd32-rs).

That upstream project provides Rust GD32 peripheral access crates generated from `svd2rust` plus a community-maintained set of GD32 SVD patches and device definitions. This repository includes a sync script at `scripts/sync-gd32-rs.sh` that copies and flattens that generated PAC code into this crate.

Because of that, this repository should not be read as though all of the PAC or register-level code was authored here. Most of the credit for the PAC content belongs to the upstream [`gd32-rs`](https://github.com/gd32-rust/gd32-rs) project and its contributors.

## What This Repository Adds

The original work in this repository is primarily the Embassy-facing integration around those PACs, including:

- re-exports and feature gating that make the synced PAC modules usable from higher-level code
- the `gd32f425` Embassy time driver in `src/time_driver_gd32f425.rs`
- supporting example code such as `examples/embassy-time-smoke.rs`
- some of the GD32F4-series-specific work involved in bringing the crates into an Embassy-oriented workflow

In other words, the main contribution here is the Embassy integration layer and the work toward a HAL built on top of the generated GD32 PAC crates, not authorship of the full PAC surface itself.

## Scope

At the moment this repository is best understood as:

- a flattened collection of GD32 PAC modules synced from `gd32-rs`
- a staging ground for Embassy support on GD32 parts
- ongoing work toward a practical Embassy-compatible HAL built on top of those PACs

## Contributions

Contributions are welcome.

Useful contributions include:

- fixes or extensions to Embassy support
- testing on real hardware
- improvements to GD32F4-series coverage and bring-up
- documentation, examples, and cleanup

Small, focused pull requests are especially helpful while the project is still taking shape.
